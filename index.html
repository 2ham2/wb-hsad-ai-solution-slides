실시간 신호 소스

검색 트렌드 API(구글 트렌드, 네이버 데이터랩 등) : 검색량 급상승 감지

날씨 API : 기온, 강수, 미세먼지 등 (예측 가능한 신호)

신호-업종 매핑 테이블

어떤 외부 신호가 어떤 업종/브랜드와 연관 있는지 정의해둔 규칙 또는 데이터
(예: "폭염" → 아이스크림/음료/여행(계곡·바다), "장마" → 배달/실내여가)

업종별 소재 템플릿/톤앤매너

신호가 감지됐을 때 바로 꽂아넣을 수 있는 카피 구조(소구축 템플릿)가 미리 준비돼 있어야 생성 속도가 실제로 빠르게 진행될 수 있음

분석 방식
1 변화율 계산

각 신호 소스를 주기적으로(예: 시간 단위) 크롤링/조회해서 전일·전주 대비 변화율 계산

2 1차 필터링

임계치(예: 검색량 전일 대비 200% 이상 증가) 넘는 키워드를 "급상승 후보"로 1차 필터링

3 업종/브랜드 매칭

신호-업종 매핑 테이블과 대조해서 관련 업종/브랜드 매칭 (데이터가 쌓이면 임베딩 유사도 매칭으로 고도화 가능)

4 카피 즉시 생성

매칭된 업종의 카피 템플릿에 해당 키워드를 삽입해 LLM으로 카피 여러 안 즉시 생성

신호와 업종이 표면적으론 연관돼 보여도 실제로 결합하면 부적절한 경우가 많음. "감지된 이슈 = 논란성/민감 이슈인지" 판별하는 안전장치가 반드시 필요

3. 숏폼 영상 대량생산
분석 방식
1 구간별 벡터화

영상을 짧은 시간 윈도우(예: 1~2초 단위)로 쪼개서 구간별 특징값(음량 변화, 키워드 등장, 장면전환 여부)을 벡터화

2 하이라이트 스코어 랭킹

각 신호에 가중치를 곱해 합산한 "하이라이트 스코어"로 전체 구간을 랭킹

3 가중치 튜닝/보정

과거 편집 이력이 있다면 실제 선택 구간과 비교해 가중치를 조정 / 없다면 감으로 정하고 피드백으로 점진적 보정

3. 숏폼 영상 대량생산
현실적인 한계와 주의할 점

맥락 이해의 한계

AI가 뽑은 구간이 수치적으로는 "하이라이트"지만 맥락상 앞뒤 내용이 끊겨 이상하게 들릴 수 있음. 광고 소재는 문제 제기 → 해결 → CTA 흐름이 중요해 단순 신호만으로는 스토리 구조를 못 읽을 수 있음

업종/포맷별 하이라이트 기준이 다름

제품 시연 영상과 인터뷰형 영상은 "좋은 구간"의 정의 자체가 다름. 하나의 범용 스코어링 로직으로 모든 소재를 커버하기 어려워 포맷별 가중치 튜닝 필요
