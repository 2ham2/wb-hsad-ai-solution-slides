실시간 신호 소스

검색 트렌드 API(구글 트렌드, 네이버 데이터랩 등) : 검색량 급상승 감지
소셜 리스닝 데이터 : 언급량이 갑자기 늘어난 키워드
날씨 API : 기온, 강수, 미세먼지 등 (예측 가능한 신호)

신호-업종 매핑 테이블

어떤 외부 신호가 어떤 업종/브랜드와 연관 있는지 정의해둔 규칙 또는 데이터
(예: "폭염" → 아이스크림/음료/여행(계곡·바다), "장마" → 배달/실내여가)

업종별 소재 템플릿/톤앤매너

신호가 감지됐을 때 바로 활용할 수 있는 소재 템플릿이
미리 준비되어 있어야 생성 속도가 실제로 빠르게 진행 가능

1 변화율 계산

각 신호 소스를 주기적으로(예: 시간 단위)
크롤링/조회해서 전일·전주 대비 변화율 계산

2 1차 필터링

임계치(예: 검색량 전일 대비 200% 이상 증가) 넘는 키워드를
"급상승 후보"로 1차 필터링

3 업종/브랜드 매칭

신호-업종 매핑 테이블과 대조해서 관련 업종/브랜드 매칭
(데이터가 쌓이면 임베딩 유사도 매칭으로 고도화 가능)

4 카피 즉시 생성

매칭된 업종의 카피 템플릿에 해당 키워드를 삽입해
LLM으로 카피 여러 안 즉시 생성

2. 업종별 데이터 기반 크리에이티브 자산화
과거에 집행했던 소재들의 비주얼 특징과 성과 데이터(CTR/CVR)를 매칭시켜서
각 업종 및 페르소나엔 어떤 비주얼 요소가 잘 먹혔는지를 패턴으로 뽑아내고, 다음 소재 생성 시 프롬프트나 체크리스트에 반영하는 구조
데이터 볼륨이 중요해서 HSAD & WB에서 이미 집행하고 있는, 집행해본 이력이 있는 업종
소재 원본 + 메타데이터 / 성과 데이터

과거 집행 소재 이미지 파일 / 매체·업종·캠페인·타겟 세그먼트 태그 / 집행 기간, 노출량

CTR, CVR, 전환당비용 등 매체 리포트 지표 — 소재별로 최소 일정 노출량 이상 확보된 것만 (노출 적은 소재는 노이즈가 큼)

경쟁사 데이터
경쟁사 최신 데이터를 함께 분석하여, 비슷한 소재만 반복적으로 뽑히는 문제점 해소

비주얼 특징 추출 (이미지에서 뽑아낼 수 있는 특징)

구성적 특징
인물 등장 여부·수, 클로즈업/전체샷 여부, 텍스트 영역 비율

색채적 특징
주조색, 채도/명도 평균, 배경색 vs 브랜드 컬러 대비

레이아웃
여백 비율, 좌우/상하 대칭성, CTA 버튼 위치

의미적 특징
단독컷인지 라이프스타일컷인지, 실사인지 일러스트인지

1 데이터 수집

소재 원본 + 매체 성과 리포트

2 비주얼 특징 추출

인물·색상·텍스트밀도·레이아웃

3 통합 저장

소재 x 특징 x 성과 테이블

4 세그먼트별 상관 분석

업종-타겟별 성과 패턴 도출

5 활용

프롬프트 반영 / 참고카드 / 신규 소재 스코어링
